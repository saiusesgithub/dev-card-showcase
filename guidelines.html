Simple rules to keep our projects collection consistent, clean, and beginner-friendly

Overview

These guidelines help maintain quality and consistency across all projects. They're meant to guide, not restrict. We're building a community where everyone can learn and share!

Core Guidelines

1

Keep Descriptions Clear

Write 1-2 lines explaining what your project does. Be concise but informative.

Good: "A weather app that shows 5-day forecasts with animations."
Avoid: "My first project... it's not very good but I tried..."
2

Learning Projects Welcome

Small projects, tutorials, and learning experiments are encouraged!

  • Beginner projects welcome
  • Tutorial implementations
  • Code experiments
3

Avoid Duplicates

Check existing projects before adding yours. Add something new or improved.

Use the search function on projects page to check for similar projects
4

Use Proper Tags

Tag your project with relevant technologies (max 5 tags).

HTML CSS JavaScript API React

Submission Checklist

Project has a clear, concise description
Both GitHub and Live Demo links are provided
Relevant tags are included (max 5)
Project is not a duplicate of existing submissions
All links are working and accessible

Project Card Format

Project Title

Brief description (1-2 lines) explaining what your project does.

HTML CSS JavaScript API
GitHub Live Demo
By Your Name
Follow this exact structure when adding your project

Quick Reference

Do's

  • Keep descriptions short and clear
  • Include both GitHub and Live links
  • Use relevant tech tags
  • Test your links before submitting
  • Add beginner-friendly projects

Don'ts

  • Don't submit incomplete projects
  • Avoid overly complex descriptions
  • Don't use offensive language
  • Avoid broken or private repos
  • Don't spam with multiple similar projects
